**2주차 시간**: 약 17h

### 2주차 핵심 학습
- **1인칭 카메라 부착** — Transform 부모-자식 관계, 카메라 위치 고정
- **마우스 룩의 두 축 분리** — yaw는 캡슐 자체 회전, pitch는 카메라만 회전 (수직 ±85도 클램프)
- **Server Authoritative 체감** — 클라는 입력만, 자기 위치도 서버 응답 기다림
- **Lerp 보간** — 비정기 스냅샷을 부드럽게 렌더링
- **Cursor.lockState** — FPS의 기본 (Locked + Visible=false)

### 2주차 마우스 룩 힌트
- yaw (좌우): `transform.Rotate(0, mouseX * sensitivity, 0)` → 캡슐 자체 회전
- pitch (상하): `cameraTransform.localEulerAngles.x = -mouseY * sensitivity` → 카메라만
- pitch는 -85 ~ +85도로 클램프 (위/아래 끝까지 가면 뒤집힘)
- 자기 캡슐 자기는 즉시 회전 적용 (서버 응답 기다리면 끔찍함), 동시에 서버에 송신

### 2주차 위험 시점
- 마우스 룩이 어색하면 → sensitivity 튜닝, pitch 클램프 점검
- 점프가 텔레포트처럼 보이면 → 보간 계수 조정 (점프는 빠르므로 Lerp 0.3 정도)
- 다른 캡슐의 회전이 안 따라오면 → S_Snapshot에 yaw 포함 확인

## 클라 담당이 자주 빠지는 함정

| 함정 | 결과 | 대응 |
|---|---|---|
| 메인 스레드 제약 무시 | 즉시 에러 | ConcurrentQueue → Update에서 처리 |
| DontDestroyOnLoad 빼먹음 | 씬 전환 시 NetworkManager 사라짐 | 1주차에 무조건 추가 |
| Cursor.lockState 빼먹음 | 마우스 화면 밖 나감, FPS 안 됨 | Game 씬 진입 시 Locked, ESC 시 None |
| pitch 클램프 누락 | 위/아래 끝까지 가면 화면 뒤집힘 | -85 ~ +85도 Mathf.Clamp |
| Shared 동기화 깜빡 | 빌드 깨짐 | 서버 담당이 패킷 추가 시 즉시 복사 |
| UI 폴리싱에 시간 낭비 | 게임플레이 늦어짐 | 5주차까지 UI 미루기 |
| 에셋 스토어 무료 에셋 | 시간 잡아먹음 | 캡슐 + Line Renderer로 충분 |
| Lerp 계수 튜닝에 빠짐 | 보간 완벽 추구 | 0.1~0.3 사이로 적당히 |
| 자기 캡슐 자기 위치도 서버 응답 기다림 | 자기 입력이 끔찍하게 느림 | 자기는 즉시 반영 + 서버에 송신 (3주차 보강 가능) |

## 절대 잊으면 안 되는 것

1. **3주차 끝 = 발표 가능 시점.** 무조건 사수.
2. **Shared 동기화 매번 확인.** 서버 담당이 패킷 추가했을 때 즉시 복사.
3. **UI는 일단 동작만.** 폴리싱은 5주차 또는 방학.
4. **본인(서버 담당)과 매주 금요일 통합 테스트.**
5. **에셋 스토어 자제.** 캡슐 + Line Renderer + 기본 UI로 5주.
6. **자동 매칭 흐름 코드를 4주차에 함부로 지우지 말 것.** 양쪽 다 동작 상태에서 점진 교체.
7. **Cursor.lockState / pitch 클램프 / DontDestroyOnLoad — FPS의 3대 함정.** 처음부터 넣을 것.
