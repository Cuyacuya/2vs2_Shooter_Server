# server/ — 서버 작업 규칙

## 담당 범위 (A)
- TCP/UDP 소켓 수신 및 송신
- 패킷 라우팅 및 핸들러
- 게임 상태머신 (로비, 방, 인게임, 라운드 판정)
- 세션 관리 (토큰 발급, 연결 해제 처리)

## 아키텍처 원칙
- 패킷 수신 → 헤더 파싱 → PacketId로 핸들러 디스패치
- 핸들러는 1개의 역할만 담당 (로그인, 이동, 공격 등 분리)
- 게임 상태는 서버가 단일 권한으로 관리 (클라이언트 신뢰 금지)

## 주의사항
- 공유 자원 접근 시 lock 사용
- 연결 해제 시 반드시 세션 정리
